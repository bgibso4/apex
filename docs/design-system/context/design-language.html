APEX Design Language

The direction lock, decided 2026-07-21 after eight exploration rounds (references/ v1→v8). This page is the framing that dictates every screen designed in Phase 1. Locked: mindset, structure, light, color model, typography. Accent parameter: dusty blue #659BC8 is the confirmed default after v7/v8 challenge rounds; violet #8A91C9 is retained as the alternate token value — final play happens in tokens.json.

The mindset — seven laws · locked

  1. A precision instrument, not a companion. APEX feels engineered, expensive, quiet. It never cheers, decorates, or fills silence. Its confidence is stillness.
  2. One light source. Depth comes from a single overhead light told twice at half strength each: a faint radial halo behind the header lights the room; a shallow vertical card gradient with a brighter top edge lights the objects. Depth sums — it never stacks.
  3. Neutral field, meaningful color. ~90% of any screen is the neutral ramp. Exactly two hues exist: the primary accent means now — the current set, the selected control, the one next action; and moss means earned — logged sets, completed work, positive deltas. Nothing is ever colored for decoration. If a third hue appears, the design is wrong.
  4. Nothing glows. Every color is desaturated for dark surfaces. If a value would vibrate against the field, lower its chroma until it sits down.
  5. The biggest number wins. Hierarchy is carried by size and weight of numerals — tabular, tight — never by color. The most important number on a screen should be findable with peripheral vision.
  6. Structure carries hierarchy. Hairline borders, uppercase tracked eyebrows, tight negative-tracked titles, generous dark space. If hierarchy needs a colored badge to work, restructure instead.
  7. One tap for the common case. Logging a set as prescribed is one tap, forever. Every redesigned flow must hold or reduce today's touch count. Sweaty hands, limited attention — the instrument must be operable blind.

The color model — two hues · locked model · open hue

TokenValueMeaning
accent.primaryPARAMETER — default #659BC8 dusty blue (confirmed v8) · alternate #8A91C9 dusty violetnow: current set, selection, primary CTA. Halo cast derives from it. Swapped by changing one token.
state.logged#6CA383 mossearned: logged sets, completions, positive deltas. The secondary hue exists so done-vs-now never shares a color — kept deliberately.
tints10% alpha derivativeschip fills, suggestion backgrounds — always derived, never hand-picked

Why two hues and not one: the single-hue cut (v6-B) was rejected — the moss "earned" color is load-bearing feedback, worth its slot in the hue budget. Why the hue is a parameter: violet vs blue is a taste call with zero structural consequence; tokens.json makes it a one-line swap, so it stays open without blocking anything.

Typography · locked

  • System stack (SF Pro); Orbitron 800 reserved for the ΛPEX wordmark only — unchanged from today
  • Screen titles 20px / 800 / −0.3px tracking · card names 14px / 700
  • Numerals always tabular-nums; hero numerals 34px / 800; values 13px / 600
  • Eyebrow labels 9–10px / 700 / uppercase / +1.1px tracking — the signature label form, kept
  • Dynamic Type respected in SwiftUI — these are the reference sizes at default scale

Open questions — resolved in Phase 1, not before building · open

  • Primary hue — blue vs violet (vs a tuned third candidate); played with once tokens.json exists
  • Motion character — the instrument's movement personality (restrained and instant vs weighted and springy); defined against real screens, may include the halo breathing during a live session
  • Liquid Glass — content cards stay painted-light (locked); system chrome (tab bar, sheets) glass vs flat is decided when the tab bar is designed
  • Block identity — training blocks lose their four hues; whether they become text designations, luminance steps, or accent-tinted markers is a Phase 1 layout decision

What this replaces

The indigo #6366f1 system documented in context/current-state-inventory.html, and the Round 1 specimens (retired). The exploration trail lives in the repo at docs/design-system/references/ v1→v8; the winning frame is v6-A "Precision Blue, two-hue", which then survived the v7/v8 challenge rounds: warm-ink (Sumi) and fog (Nord) field casts rejected in favor of the neutral-cool field; seal-clay, weathered-iron, forged-steel, and violet-on-fog accents rejected in favor of the dusty blue. Single-hue (v6-B), pure monochrome (Quiet Professional), and glacial (Summit) were eliminated earlier.
