3Known debt the redesign fixes

  • No shared component-style layer — 33 separate StyleSheet.create blocks across screens and components, each hand-rolled from scratch. No shared style primitives exist between them, so the same visual pattern (a card, a row, a section label) is re-implemented with small drifts every time it's needed.
  • ~10 hardcoded color escapes outside the token system — the pain-severity gradient palette is duplicated verbatim in two places (PainFollowUp.tsx and running.tsx); resource-link brand colors are hardcoded in exercise/[id].tsx (YouTube red, Instagram pink, TikTok white); SupersetGroup and WeekRow use raw alpha-suffixed hex literals (e.g. #6366f150); Settings renders its integration badges with raw rgba() literals; ProgramTimeline uses raw rgba(255,255,255,…) for its white-alpha states.
  • Section-label drift — the uppercase "eyebrow" label pattern isn't backed by one token: font size varies 11px vs 10px and letter-spacing varies 0.5 vs 1.0 depending on which screen or component wrote it, so labels that are supposed to read as the same typographic voice don't quite match side by side.

4Screen census

  • (tabs)/indexHome — active program dashboard: week/block context, today's session card, health summary, PR/pain prompts.
  • (tabs)/workoutWorkout — thin render layer over useWorkoutSession; the live set-logging flow, warmup, adjust, and post-session summary all compose here.
  • (tabs)/progressProgress — estimated 1RM trend charts, volume, and a time-range selector.
  • (tabs)/runningRunning — the swappable 4th tab: pain tracking, run logging, and trends across Log / Trends sub-tabs.
  • libraryLibrary Modal — program catalog; tap a card to select, floating button activates it (activation is inline here, not a separate screen).
  • exercisesExercises — every exercise (library + user-created), grouped by muscle group, collapsible groups expanded by default.
  • historyHistory — all completed sessions grouped by program, newest first.
  • settingsSettings — training preferences, integrations, and data management.
  • program-completeProgram-complete — hosts the completion celebration takeover, then the program summary view.
  • exercise/[id]Exercise detail — metric trend chart with block bands, time-range chips, hero stat, compact session history rows; type-adaptive (weight+reps, duration, distance+time, etc.).
  • exercise/progressionExercise progression — full progression history for one exercise, reached via "View all" from the detail screen.
  • exercise/sessionsExercise sessions — full session history for one exercise, reached via "View all N sessions" from the detail screen.
  • session/[id]Session detail — read-only view of a completed past workout.
